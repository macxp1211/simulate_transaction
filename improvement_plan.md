# 多智能体模拟撮合系统改进计划

## 一、现状分析

### 已实现的核心功能
- 基础撮合引擎（价格优先、时间优先）
- 队列模拟（queue_length, queue_position）
- 逐笔成交驱动队列消耗
- T+1 账户管理（冻结/解冻）
- A股费用模型（佣金、印花税、过户费）
- 5类基础参与者（做市商、趋势、均值回归、噪声、激进）
- SQLite 持久化
- REST API + WebSocket
- 前端委托终端 + 监控面板

### 缺失的关键功能（按优先级排序）

#### P0 - A股真实交易规则（当前最影响真实感）
1. **涨跌停限制**：主板10%、ST 5%、科创板/创业板20%，基于昨收价计算
2. **价格笼子**：买入申报价不得高于基准价102%，卖出不得低于98%（沪深主板2023年新规）
3. **最小价格变动单位**：0.01元
4. **每手100股**：数量必须是100的倍数
5. **昨收价基准**：所有价格限制都需要昨收价作为参考

#### P1 - 参与者智能化（多智能体核心）
1. **算法交易者**：TWAP/VWAP 拆单执行，隐藏交易意图
2. **止损/止盈交易者**：设置触发价格，条件触发后自动报单
3. **订单簿感知交易者**：根据订单簿深度、不平衡度、流动性决策
4. **信息驱动交易者**：对价格冲击/新闻反应，产生波动聚集
5. **冰山订单参与者**：大单拆分，只显示部分数量
6. **参与者P&L跟踪**：每个参与者有自己的虚拟账户，盈亏可追踪

#### P2 - 行情生成真实感
1. **波动聚集**：GARCH-like 波动率模型，大波动后更可能有大波动
2. **订单流不平衡**：买盘/卖盘力量不平衡驱动价格方向
3. **价格冲击模型**：大单成交后的价格漂移
4. **开盘跳空**：基于隔夜信息模拟开盘价格跳空
5. **成交量分布**：不同时间段成交量分布不同

#### P3 - API体验优化
1. **延迟模拟**：不同订单类型的延迟差异（市价<限价<大宗）
2. **订单簿增量推送**：只推送变化部分，而不是全量快照
3. **API限流**：防止过度请求
4. **订单状态流推送**：从提交到成交的全生命周期推送
5. **更真实的错误信息**：资金不足、价格笼子、涨跌停等具体原因

#### P4 - 监控与分析增强
1. **订单流指标**：买卖压力比、订单簿不平衡度
2. **Spread 分析**：实时价差、深度加权价格
3. **参与者P&L面板**：谁赚了谁亏了
4. **滑点分析**：实际成交价 vs 预期价格
5. **深度图**：订单簿深度可视化

#### P5 - 前端体验
1. 深度图（Depth Chart）
2. 成交量分布柱状图
3. 买卖力量对比条
4. 参与者行为可视化

## 三、关键设计决策

### 昨收价管理
- 每个 symbol 维护一个 `previous_close` 价格
- 启动时如果没有昨收价，使用 base_price 作为默认
- 监控端可以配置目标价格，同时自动计算涨跌停范围

### 价格笼子实现
- 基准价格 = 最优买价（买入时）或 最优卖价（卖出时）
- 买入价 ≤ 基准价 × 102%
- 卖出价 ≥ 基准价 × 98%
- 如果不存在对手盘，使用最新成交价作为基准

### 涨跌停实现
- 主板：±10%（ST ±5%）
- 科创板/创业板：±20%
- 北交所：±30%
- 集合竞价阶段：无涨跌停限制（预留框架）

### 参与者P&L
- 每个参与者有自己的虚拟账户（cash, position, pnl）
- 成交时更新虚拟账户
- 监控端显示每个参与者的盈亏

### 算法交易者
- TWAP：时间加权平均价格，将大单拆分成小单在时间段内均匀执行
- VWAP：成交量加权平均价格，根据历史成交量分布拆单
- 隐藏意图：使用冰山订单，只显示部分数量

### 订单簿感知交易者
- 计算订单簿不平衡度：`(bid_depth - ask_depth) / (bid_depth + ask_depth)`
- 当不平衡度 > 阈值，预期价格上涨，买入
- 当不平衡度 < -阈值，预期价格下跌，卖出

### 波动聚集
- 使用简单 EWMA 波动率模型
- 当前波动率 = 0.94 × 上一期波动率 + 0.06 × 当前收益²
- 波动率影响参与者的订单价格和数量

## 四、接口变更

### 新增API
- `GET /api/v1/market/rules/{symbol}`：获取某标的的交易规则（涨跌停、价格笼子范围）
- `GET /api/v1/analytics/order_flow/{symbol}`：订单流分析指标
- `GET /api/v1/analytics/participants/pnl`：参与者P&L排名
- `GET /api/v1/analytics/depth/{symbol}`：深度图数据

### 修改API
- `POST /api/v1/orders`：增加更详细的拒绝原因（价格笼子、涨跌停、最小变动等）
- WebSocket：增加订单簿增量推送（`delta` 类型）
- WebSocket：增加订单状态流推送（`order_status` 类型）

### 新增配置
- 监控端：显示当前涨跌停范围、价格笼子范围
- 监控端：配置昨收价、市场类型（主板/科创板/创业板）
- 监控端：显示参与者P&L排名
