# 撮合逻辑详细说明

## 1. 核心撮合原则

本系统模拟撮合严格遵循交易所的撮合规则：

### 1.1 价格优先原则
- **买入委托**：价格高的优先成交
- **卖出委托**：价格低的优先成交

### 1.2 时间优先原则
- 同一价格层级内，先进入队列的订单优先成交

### 1.3 队列位置记录
- 委托进入队列时，记录当前队列总长度 (`queue_length`)
- 记录该订单在队列中的位置 (`queue_position`)
- 这两个字段对于分析订单成交等待时间至关重要

## 2. 撮合场景详解

### 场景 1：买入委托价格 >= 最优卖价（立即撮合）

```
市场状态:
  卖盘: 10.50 -> [OrderA: 500, OrderB: 300]
        10.51 -> [OrderC: 200]

新委托: Buy 10.52, 1000

撮合过程:
  1. 10.50 价格成交: 500 (OrderA 全部成交)
     剩余: 500
  2. 10.50 价格成交: 300 (OrderB 全部成交)
     剩余: 200
  3. 10.51 价格成交: 200 (OrderC 部分成交)
     剩余: 0

结果:
  - 新委托全部成交，成交均价 = (500*10.50 + 300*10.50 + 200*10.51) / 1000 = 10.502
  - 不产生队列记录
```

### 场景 2：买入委托价格 < 最优卖价（进入队列）

```
市场状态:
  买盘: 10.48 -> [OrderX: 1000, OrderY: 500]  queue_length=2
        10.47 -> [OrderZ: 200]                   queue_length=1
  卖盘: 10.50 -> [OrderA: 500]                   queue_length=1

新委托: Buy 10.48, 800

处理过程:
  1. 最优卖价 = 10.50
  2. 委托价格 10.48 < 10.50，不交叉
  3. 进入买盘 10.48 价格层级
  4. 记录 queue_length = 3 (OrderX + OrderY + 本订单)
  5. 记录 queue_position = 3 (前面有 2 个订单)
  6. 订单状态: QUEUED

结果:
  - 订单进入队列等待
  - 当后续逐笔成交价 >= 10.48 时，可能触发队列消耗
```

### 场景 3：逐笔成交触发队列消耗

```
队列状态（买入方）:
  10.48 -> [OrderX: 1000(pos=1), OrderY: 500(pos=2), NewOrder: 800(pos=3)]

新逐笔成交: Trade 10.48, 1200, Sell-initiated (卖方主动成交)

队列消耗过程:
  1. 成交价 10.48 >= 队列价格 10.48，触发该价格层级消耗
  2. 按 FIFO 顺序消耗:
     - OrderX: 1000 全部成交，剩余 200
     - OrderY: 500 部分成交 200，剩余 300
  3. NewOrder: 未轮到，仍在队列中

结果:
  - OrderX: FILLED, 成交价 10.48
  - OrderY: PARTIAL, 已成交 200, 剩余 300
  - NewOrder: QUEUED, 前面剩 1 个订单 (OrderY 剩余 300)
```

### 场景 4：多价格层级消耗

```
队列状态（买入方）:
  10.50 -> [OrderA: 100]   queue_length=1
  10.49 -> [OrderB: 200]   queue_length=1
  10.48 -> [OrderC: 300]   queue_length=1

新逐笔成交: Trade 10.50, 500, Sell-initiated

消耗过程:
  1. 成交价 10.50 >= 10.50，触发
  2. 成交价 10.50 >= 10.49，触发
  3. 成交价 10.50 >= 10.48，触发
  4. 按价格从高到低、同一价格内 FIFO 消耗:
     - OrderA: 100 全部成交，剩余 400
     - OrderB: 200 全部成交，剩余 200
     - OrderC: 200 部分成交，剩余 100

结果:
  - 10.50 价格层级清空
  - 10.49 价格层级清空
  - 10.48 价格层级剩余 100
```

## 3. 逐笔成交方向判断

逐笔成交数据通常包含以下信息来判断主动方：

```python
class TradeEvent:
    price: Decimal
    quantity: int
    
    # 方向判断（根据数据源不同）
    direction: str  # "buy" = 买方主动（外盘）, "sell" = 卖方主动（内盘）
    
    # 或者通过成交价格和盘口的关系判断:
    # - 成交价 >= 最优卖价 → 买方主动（抢吃卖方挂单）
    # - 成交价 <= 最优买价 → 卖方主动（砸穿买方挂单）
    # - 成交价在买卖价之间 → 中性成交（双方同时下单撮合）
```

## 4. 队列长度动态变化

队列长度不是静态的，而是随时间动态变化的：

```
时间线:
  T1: 买盘 10.48 -> [OrderX]                    queue_length=1
  
  T2: 买盘 10.48 -> [OrderX, OrderY]            queue_length=2
  
  T3: 逐笔成交 10.48, OrderX 全部成交
      买盘 10.48 -> [OrderY]                    queue_length=1
      
  T4: 新委托 Buy 10.48 进入队列
      买盘 10.48 -> [OrderY, NewOrder]          queue_length=2
      NewOrder.queue_position = 2
      
  T5: 逐笔成交 10.48, OrderY 全部成交
      买盘 10.48 -> [NewOrder]                  queue_length=1
      NewOrder 前面剩 0 个订单
```

## 5. 特殊情况处理

### 5.1 涨停/跌停价委托

- **涨停价买入**：买入价格 = 涨停价，所有卖盘价格 <= 涨停价，应全部立即成交
- **跌停价卖出**：卖出价格 = 跌停价，所有买盘价格 >= 跌停价，应全部立即成交

### 5.2 市价委托

- 市价买入：视为最优卖价委托，立即全部成交
- 市价卖出：视为最优买价委托，立即全部成交

### 5.3 部分成交后的队列处理

当订单部分成交后，剩余部分继续留在队列中：
- 更新 `filled_qty`
- 更新 `remaining_qty`
- 保持 `queue_position`（相对位置不变）
- 状态变为 `PARTIAL`

### 5.4 撤单处理

从队列中移除订单时：
1. 找到订单所在价格层级
2. 从队列中移除
3. 更新该价格层级的 `queue_length`
4. 后续订单的相对位置自动前移（通过索引计算）

## 6. 撮合精度说明

本系统模拟撮合的精度来源于：

1. **逐笔级驱动**：每个逐笔成交事件都触发一次队列扫描
2. **真实价格对齐**：成交价、盘口价都使用真实 Level-2 数据
3. **队列位置精确**：记录进入队列时的精确位置
4. **时间戳精确**：使用行情源的时间戳（微秒/毫秒级）

撮合结果与真实交易所的差异主要来源：
- 真实的券商报单通道延迟（本系统模拟理想情况）
- 真实的行情传输延迟
- 部分数据源不包含逐笔成交的主动方向，需要推断
